# Product Brief — Tomboj Studio

Tomboj Studio is a private AI-assisted content operations system designed to manage multi-brand LinkedIn workflows from one daily operating surface.

## Purpose

The purpose of Tomboj Studio is to reduce the operational drag of running multiple content identities while preserving human judgment at the moments that matter.

The system is designed around a short daily workflow:

1. Open Studio.
2. See what needs attention.
3. Approve, redirect, reroll, or confirm.
4. Review performance signals.
5. Close the tab.

## Problem

AI can generate content quickly, but content operations still break down when there is no shared system of record.

Common issues include:

* Drafts scattered across tools
* No clear state for each content asset
* Revision history lost in chats
* AI feedback not captured structurally
* Content calendars disconnected from analytics
* Manual tracking across multiple identities
* Unclear ownership between human and AI agent

Tomboj Studio is designed to solve the operating layer around AI-assisted content creation.

## Product Thesis

AI content systems need more than prompting. They need state, memory, approval gates, observability, and performance feedback loops.

The core product thesis:

> The human should not manage every task. The human should manage judgment, direction, and approval.

## Primary User

The primary user is an operator managing multiple content identities with AI assistance.

The system is designed for someone who needs:

* Daily clarity
* Low-friction approvals
* Structured revision history
* Visibility across brands
* Analytics connected to content decisions
* A trusted surface for AI-agent collaboration

## V1 Scope

V1 focuses on:

* Content cards
* Board workflow
* Today view
* Calendar view
* Dashboard
* Revision history
* Thread notes
* Agent action trace
* Manual publish confirmation
* LinkedIn analytics import

V1 does not include:

* Automated publishing
* Engagement workflows
* Push notifications
* Multi-user administration
* Public SaaS access
* Open-source distribution

## Design Principle

Studio is a database with a good face.

Agents can work in the system continuously. The human should only need to look briefly, understand the state of the system, and make the few decisions that require judgment.
